### 指示
電子カルテシステムの過去カルテ領域のスクリーンショットです。$image_desc
以下の2つのタスクを同時に処理してください。

---

### タスク1: 最後エントリの補完
「取得済みカルテデータ」の最後のエントリ（日付: $last_date）が、
前の画面で途中から切れている可能性があります。
画像・OCRテキストを参照し、続きがあれば continuation に返してください。
- 取得済みカルテデータの最後のエントリ末尾の直後から続く部分のみを返す（取得済みの内容自体は含めない）
- 新しい日付のエントリのテキストは含めない
- 続きがない場合は continuation に空文字列を返す

---

### タスク2: 新規エントリの抽出
「取得済みカルテデータ」に含まれない新しい日付のエントリを画像・OCRテキストから逐語的に抽出してください。
- 画像に表示されているテキストのみを逐語的にコピー
- 医学的に「あるべき内容」を補完・推測することを禁止
- 取得済みカルテデータに含まれる内容は new_entries に含めない
- 本文が「TEST」など短い場合でも、その短い内容をそのまま記録すること
- 新しいエントリがない場合は空配列 [] を返す

---

### 取得済みカルテデータ（参照用・重複判定に使用）
```json
$accumulated_chart
```

### このシステムの表示レイアウト
- 各診療録エントリは「本文テキスト」の直後（下部）に「日付・著者スタンプ」（例: `2026年04月10日 15:51 内科 診療記録`）が表示されます
- 日付スタンプの直上に書かれているテキストが、その日付に属する本文です
- 日付スタンプと本文の対応は画像の空間的な位置関係で判断してください

### 過去のスクロールで蓄積されたOCR認識結果（参考）
```
$prev_ocr_text
```

### 今回のスクロール画像のOCR認識結果（参考）
```
$ocr_text
```

### 厳守事項
- 画像・OCR認識テキストに存在しないテキストを生成しないでください（ハルシネーション防止）
- 取得済みカルテデータに既に含まれている内容を continuation や new_entries に重複して含めないでください

### 出力フォーマット（JSONのみ）
{
  "continuation": "続きのテキスト（なければ空文字列）",
  "new_entries": [
    {
      "date": "YYYY年MM月DD日",
      "content": "本文テキスト（画像から逐語的にコピー）"
    }
  ]
}
