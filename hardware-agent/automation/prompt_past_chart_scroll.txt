### 指示
電子カルテシステムの過去カルテ領域のスクリーンショットです。$image_desc
以下の2つのタスクを同時に処理してください。

---

### タスク1: 最後エントリの補完
取得済みカルテデータの最後のエントリ（日付: $last_date、種別: $last_record_type）の本文が
前の画面で途中から切れている可能性があります。

以下の手順で continuation を判定してください:
1. 現在の画像・OCR テキストの中に「$last_date」かつ「$last_record_type」を含むスタンプを探してください
2. そのスタンプが見つかった場合: スタンプの直上にある本文テキストのうち、
   取得済みカルテデータ末尾の直後に続く部分のみを continuation に返してください
3. そのスタンプが見つからない場合、または続きがない場合: 空文字列を返してください
- 他の種別のスタンプ（例: 看護記録、チーム記録）の直上にあるテキストは continuation に含めないでください
- 新しい日付のエントリのテキストは含めないでください

---

### タスク2: 新規エントリの抽出
「取得済みカルテデータ」に含まれない新しい日付のエントリを画像・OCRテキストから逐語的に抽出してください。
- 画像に表示されているテキストのみを逐語的にコピー
- 医学的に「あるべき内容」を補完・推測することを禁止
- 取得済みカルテデータに含まれる内容は new_entries に含めない
- 本文が「TEST」など短い場合でも、その短い内容をそのまま記録すること
- 各エントリの record_type は日付スタンプから抽出してください（例: 「診療記録」「看護記録」「チーム記録」）
- 以下のエントリは new_entries に含めないでください:
  - 処方・注射オーダーフォーム（「点滴注射」「内服薬」等の薬剤名・容量・投与方法のみが本文を構成するもの）
  - 検査オーダーフォーム（「管電圧:」「撮影部位:」「方向数:」等の検査パラメータのみが本文を構成するもの）
  - 「【 有症状指示 】」で始まるもの（看護師向け症状時指示）
  - 「【 入院その他の指示 】」で始まるもの（看護師向け入院指示）
  ※ 医師が検査所見・結果を記述した SOAP 記録（例: "胸部X線で..."、"血液検査でWBC..."）は除外しない
- 新しいエントリがない場合は空配列 [] を返す

---

### 取得済みカルテデータ（参照用・重複判定に使用）
```json
$accumulated_chart
```

### このシステムの表示レイアウト
- 各診療録エントリは「本文テキスト」の直後（下部）に「日付・著者スタンプ」（例: `2026年04月10日 15:51 内科 診療記録`）が表示されます
- 日付スタンプの直上に書かれているテキストが、その日付に属する本文です
- 日付スタンプと本文の対応は画像の空間的な位置関係で判断してください

### 過去のスクロールで蓄積されたOCR認識結果（参考）
```
$prev_ocr_text
```

### 今回のスクロール画像のOCR認識結果（参考）
```
$ocr_text
```

### 厳守事項
- 画像・OCR認識テキストに存在しないテキストを生成しないでください（ハルシネーション防止）
- 取得済みカルテデータに既に含まれている内容を continuation や new_entries に重複して含めないでください

### 出力フォーマット（JSONのみ）
{
  "continuation": "続きのテキスト（なければ空文字列）",
  "new_entries": [
    {
      "date": "YYYY年MM月DD日",
      "record_type": "診療記録",
      "content": "本文テキスト（画像から逐語的にコピー）"
    }
  ]
}
