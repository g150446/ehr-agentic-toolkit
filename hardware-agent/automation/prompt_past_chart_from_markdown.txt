### 指示
電子カルテの過去カルテ画像をマークダウンに変換した2つのテキストを受け取ります。
以下の2つのタスクを同時に処理してください。

---

### タスク1: 最後エントリの補完
取得済みカルテデータの最後のエントリ（日付: $last_date、種別: $last_record_type）の本文が
前の画面で途中から切れている可能性があります。

以下の手順で continuation を判定してください:
1. 今回のマークダウンで、最初の ## ヘッダーより前に出現するテキストが continuation 候補です
2. 前回のマークダウンの末尾内容と照合し、今回のマークダウン先頭テキストが $last_record_type の続きであれば continuation に返してください
3. 今回のマークダウンが最初から ## ヘッダーで始まる場合、または続きがない場合: 空文字列を返してください
- 他の種別（例: 看護記録、チーム記録）のヘッダー直下のテキストは continuation に含めないでください
- 新しい日付のエントリのテキストは含めないでください
- 「[済]」を含む行、「■ ...■■」等の形式で始まる培養・検査オーダーセット名行とその後続の検査名列挙（塗抹鏡検・微培養同定・薬剤感受性など）、「点滴注射」「内服薬」等が文章の中でなく**単独の見出し行**として現れその直後に薬剤名・容量・投与方法が列挙されるオーダーフォーム形式の行群は continuation に含めないこと
  ※ 医師の SOAP 記録の文中に薬剤名が登場する場合（例:「CTRX を開始」「セフトリアキソン投与」）は除外しない

---

### タスク2: 新規エントリの抽出
「取得済みカルテデータ」に含まれない新しい日付のエントリを今回のマークダウンから逐語的に抽出してください。
- マークダウンに表示されているテキストのみを逐語的にコピー
- 医学的に「あるべき内容」を補完・推測することを禁止
- 取得済みカルテデータに含まれる内容は new_entries に含めない
- 本文が「TEST」など短い場合でも、その短い内容をそのまま記録すること
- 各エントリの record_type は ## ヘッダーから抽出してください（例: 「診療記録」「看護記録」「チーム記録」）
  - ## ヘッダーに記録種別テキスト（「診療記録」「看護記録」「チーム記録」等）が明示されていない場合（例: `[2026.05.18 15:47 金谷 徳和]` のような日付・時刻・人名のみの形式）は、そのエントリを new_entries に含めないでください（record_type を本文から推測しないこと）
  - 上記に該当するエントリで、本文に取得済みカルテデータに含まれていない医療情報（バイタルサイン・身体所見等）がある場合は、その内容（オーダー行を除く）を continuation として追加してください
- 各エントリの time は ## ヘッダーから時刻（HH:MM 形式）を抽出してください。時刻が不明な場合は空文字列。
  - ただし、エントリの本文がオーダー内容のみ（処方・検査・培養等の名称列挙）で構成されている場合は、ヘッダーに時刻があっても time を空文字列にしてください（この時刻はオーダー受付時刻であり診療記録の作成時刻ではないため）
- 以下のエントリは new_entries に含めないでください:
  - 処方・注射オーダーフォーム（「点滴注射」「内服薬」等の薬剤名・容量・投与方法のみが本文を構成するもの）
  - 検査オーダーフォーム（「管電圧:」「撮影部位:」「方向数:」等の検査パラメータのみが本文を構成するもの）
  - 細菌培養・微生物検査オーダー（「■ ... ■■」形式のオーダーセット名と「塗抹鏡検」「微培養同定」「薬剤感受性」等の検査名のみが本文を構成するもの）
  - 「【 有症状時指示 】」または「【 有症状指示 】」で始まるもの（看護師向け症状時指示）
  - 「【 入院その他の指示 】」で始まるもの（看護師向け入院指示）
  ※ 医師が検査所見・結果を記述した SOAP 記録（例: "胸部X線で..."、"血液検査でWBC..."）は除外しない
- マークダウンのテーブル形式（| ラベル | 内容 |）から抽出する場合は、"内容"列のテキストのみを content に含めること（パイプ記号・ヘッダー行・区切り行は含めない）
  - 「S」「O」「A」「P」「自由」「主」「#0」「#1」等のラベル列の値は content に含めないこと
  - 誤り例: `"自由 体調変わり:あり"` → 正しくは `"体調変わり:あり"`
- 「画像下部で切れているため」「原文通り」「以下省略」などの注釈テキストは content に含めないこと
- 新しいエントリがない場合は空配列 [] を返す

---

### 取得済みカルテデータ（参照用・重複判定に使用）
```json
$accumulated_chart
```

### 前回のマークダウン（参考）
```
$prev_markdown
```

### 今回のマークダウン
```
$markdown
```

### 厳守事項
- マークダウンに存在しないテキストを生成しないでください（ハルシネーション防止）
- 取得済みカルテデータに既に含まれている内容を continuation や new_entries に重複して含めないでください

### 出力フォーマット（JSONのみ）
{
  "continuation": "続きのテキスト（なければ空文字列）",
  "new_entries": [
    {
      "date": "YYYY年MM月DD日",
      "time": "HH:MM",
      "record_type": "診療記録",
      "content": "本文テキスト（マークダウンから逐語的にコピー）"
    }
  ]
}
